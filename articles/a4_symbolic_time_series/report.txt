Symbolic Time Series for Financial Analysis: 3 Methods

Objective

Financial time series are noisy, high-frequency, and difficult to interpret directly.
Pattern mining requires discrete states.
Text-processing tools (like regex) operate on symbolic sequences, not numbers.
Therefore, we must transform continuous price data into symbolic representations.

How does the choice of symbolic encoding affect the structure and statistical properties of financial time series?

2. Theoretical Background

2.1 What Is Symbolic Time Series Analysis?

4. Encoding Method 1 — Directional Encoding (Baseline)
Method:

Define simple states:

U = positive return above threshold
D = negative return below threshold
S = small movement

Threshold could be:

Fixed (e.g., ±0.5%)
Or volatility-adjusted

What It Captures:

Momentum
Run-length
Reversals

What It Discards:

Magnitude beyond threshold
Volatility structure
Distribution shape

Include:

Example transformation
Resulting symbol string excerpt

5. Encoding Method 2 — Quantile-Based Encoding

This is likely your strongest method.

Method:

Divide return distribution into equal-frequency bins (e.g., 5 bins).
Assign symbols A–E.

Example:

A = bottom 20%
E = top 20%

Why This Is Strong:

Adapts to fat tails
Avoids Gaussian assumptions
Ensures balanced symbol frequencies

Analysis:

Symbol distribution
Entropy comparison
State transition matrix

Key insight:
Quantile encoding produces statistically stable symbolic distributions.

6. Encoding Method 3 — SAX (Shape-Based Encoding)

Explain clearly but concisely:

Steps:

Normalize (z-score)
Apply Piecewise Aggregate Approximation (PAA)
Use Gaussian breakpoints to assign symbols

Important Discussion:

Gaussian breakpoint assumption
What happens with fat-tailed returns?
Does symbol frequency remain balanced?
This is where you critically evaluate theory vs. financial reality.

8. What This Means for Pattern Mining

Explain:

Regex requires discrete symbols.
Encoding determines what patterns are even possible.
Poor encoding → meaningless patterns.
Good encoding → interpretable structural signals.

Example:

Directional encoding → good for run detection
Quantile encoding → good for extremal-event analysis
SAX → good for macro-shape matching

9. Key Insights

Strong, concise takeaways:
Financial time series are not Gaussian.
Quantile encoding handles heavy tails better.
Encoding choice drastically changes statistical structure.
Symbolisation is not neutral — it embeds assumptions.
